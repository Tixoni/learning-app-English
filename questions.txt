# ЧАСТЬ I. Теоретические вопросы

# HTML, JS — основы языков

Почему HTML называют языком разметки, а JavaScript — языком программирования, и в чём принципиальное различие их задач на странице? | HTML описывает структуру и содержимое страницы с помощью тегов (разметки), не содержит логики и не выполняется как инструкции. JavaScript — полноценный язык программирования: содержит переменные, условия, циклы, функции и управляет поведением страницы динамически | HTML управляет стилями элементов | JavaScript описывает структуру страницы | HTML и JavaScript выполняют одинаковые задачи

Назовите два способа подключить JavaScript к странице и объясните, чем подключение внешнего JS-файла отличается от подключения CSS | Первый — встроенный скрипт через тег <script>код</script>; второй — внешний файл через <script src="file.js"></script>. Отличие от CSS: у <link rel="stylesheet"> нет атрибута src, а у <script> нет rel; кроме того, <script> блокирует парсинг страницы, а CSS подключается параллельно | Первый — через атрибут onclick; второй — через тег <style> | JS подключается через тег <link>, а CSS — через <script> | Оба языка подключаются одинаково через тег <meta>

Что такое «операнд», каков приоритет арифметических операторов и зачем нужен оператор группировки? | Операнд — это значение, над которым выполняется операция (например, в 2 + 3 операнды — 2 и 3). Приоритет: унарный минус > * / % > + -. Оператор группировки () повышает приоритет выражения внутри, заставляя вычислить его первым | Операнд — это оператор действия; приоритет всегда слева направо | Оператор группировки используется только для строк | Операнды — это ключевые слова языка

Что возвращает оператор взятия остатка % и в какой типовой задаче он применяется? | Оператор % возвращает остаток от деления одного числа на другое. Типовая задача — проверка чётности числа: число % 2 === 0 означает, что число чётное | Оператор % вычисляет процент от числа | Оператор % возвращает целую часть от деления | Оператор % используется только для строк

Что такое конкатенация и какого типа получится результат при сложении числа со строкой? | Конкатенация — объединение строк через оператор +. При сложении числа со строкой (например, 1 + "2") JavaScript автоматически преобразует число в строку и результат будет строкой "12". При сложении двух чисел результат будет числом | Конкатенация — умножение строк; число + строка дают число | При сложении числа со строкой возникает ошибка TypeError | Конкатенация применяется только к массивам

Чем шаблонные строки отличаются от обычных строк в кавычках и какие две дополнительные возможности они дают? | Шаблонные строки заключаются в обратные кавычки (`). Две дополнительные возможности: 1) интерполяция выражений через ${выражение}; 2) многострочные строки без \n — перенос строки внутри шаблонной строки сохраняется | Шаблонные строки быстрее обычных по производительности | Шаблонные строки нельзя присвоить переменной | Шаблонные строки используются только в TypeScript

В чём разница между объявлением переменной и присваиванием ей значения, и почему ключевое слово let указывают только один раз? | Объявление переменной — создание имени в памяти (let x), присваивание — запись значения (x = 5). Их можно совместить: let x = 5. Ключевое слово let указывают только один раз, потому что повторное объявление той же переменной вызовет ошибку SyntaxError — переменная уже существует в этой области видимости | let можно писать перед каждым присваиванием | Объявление и присваивание — это одно и то же | Переменную нужно объявлять заново при каждом изменении значения

Что такое нотация camelCase и каким требованиям должно отвечать хорошее имя переменной? | camelCase — стиль написания составных слов: первое слово строчными, каждое последующее с заглавной буквы (например, myVariableName). Хорошее имя: описательное (отражает смысл), начинается с буквы или _, не содержит пробелов и спецсимволов, не совпадает с зарезервированными словами | camelCase — все буквы заглавные | Хорошее имя переменной должно быть как можно короче, в идеале одна буква | Имя переменной может начинаться с цифры

Что такое операторы сравнения == и === и почему на практике рекомендуют строгое сравнение? | == (нестрогое) сравнивает значения с приведением типов: "5" == 5 даёт true. === (строгое) сравнивает и значение, и тип без приведения: "5" === 5 даёт false. На практике рекомендуют === чтобы избежать неочевидных ошибок из-за автоматического приведения типов | == всегда быстрее === | === сравнивает только типы, игнорируя значения | == строже === и проверяет и тип, и значение

Опишите поведение логических операторов && и || и НЕ (!) и укажите, при каких значениях && даёт true, а || — false | && (И) возвращает true только когда оба операнда true; иначе false. || (ИЛИ) возвращает false только когда оба операнда false; иначе true. ! (НЕ) инвертирует булево значение. && использует «короткое замыкание»: если первый операнд false, второй не вычисляется | && возвращает true если хотя бы один операнд true | || возвращает false если хотя бы один операнд true | ! применяется только к числам

# Типы данных, массивы, управляющие конструкции

Какие значения в JavaScript считаются «ложными» (falsy) и как конструкция if интерпретирует не-булево значение? | Falsy-значения: false, 0, "" (пустая строка), null, undefined, NaN. Конструкция if автоматически приводит любое значение к булевому: falsy-значения работают как false, все остальные (truthy) — как true | Falsy-значения: false, 0, пустой массив [] и пустой объект {} | Конструкция if работает только с настоящими булевыми значениями | Falsy — это только false и null

Что такое массив, с какого индекса начинается нумерация его элементов и какое свойство возвращает их количество? | Массив — упорядоченная коллекция элементов, хранящихся под числовыми индексами. Нумерация начинается с 0. Количество элементов возвращает свойство length (например, arr.length) | Нумерация массива начинается с 1 | Количество элементов возвращает метод arr.count() | Массив хранит только однотипные данные

Сравните циклы for, while и do...while и объясните, чем do...while принципиально отличается от while | for — цикл со счётчиком, используется когда число итераций известно заранее. while — проверяет условие перед каждой итерацией, тело может не выполниться ни разу. do...while — проверяет условие после итерации, поэтому тело выполнится хотя бы один раз — в этом принципиальное отличие | do...while работает быстрее while | for можно использовать только с массивами | while проверяет условие после итерации

Что такое функция и в чём разница между её параметрами и аргументами при вызове? | Функция — именованный блок кода, который можно вызывать многократно. Параметры — это переменные, объявленные в определении функции (function foo(a, b)). Аргументы — конкретные значения, передаваемые при вызове (foo(1, 2)). Параметры — это «слоты», аргументы — то, что в них попадает | Параметры и аргументы — синонимы | Аргументы объявляются в теле функции | Функция не может принимать более двух параметров

Каково назначение оператора return, что вернёт функция без него и как return влияет на выполнение функции? | return завершает выполнение функции и возвращает указанное значение вызывающему коду. Функция без return (или с пустым return) возвращает undefined. После выполнения return функция немедленно прекращает работу — код после него не выполняется | return можно использовать только в конце функции | Функция без return возвращает null | return только прерывает выполнение, но не возвращает значение

Что такое объект в JavaScript, чем он отличается от массива и в каких случаях его удобнее использовать? | Объект — это коллекция пар «ключ: значение», где ключи — строки (или Symbol). Массив — упорядоченная нумерованная коллекция. Объект удобен, когда данные имеют названные поля (user.name, user.age), а не числовой порядок | Объект — это массив с именованными индексами, они идентичны | Массив хранит пары ключ-значение, а объект — числовые индексы | Объект нельзя вкладывать в массив

Перечислите примитивные типы данных и объясните назначение оператора typeof | Примитивные типы: string, number, boolean, null, undefined, symbol, bigint. Оператор typeof возвращает строку с названием типа значения (typeof 42 → "number", typeof "hi" → "string"). Исторический баг: typeof null возвращает "object" | Примитивные типы: string, number, array, object | typeof возвращает числовой код типа | typeof определяет только числа и строки

В чём смысловое различие между undefined и null? | undefined означает, что переменная объявлена, но ей не присвоено никакого значения — это «отсутствие значения по умолчанию». null — явно установленное программистом «пустое» значение, означающее намеренное отсутствие объекта. undefined появляется автоматически, null — устанавливается разработчиком | undefined и null — полные синонимы | null появляется автоматически, undefined устанавливается вручную | undefined означает ошибку в коде

Что такое тип Symbol и какое его главное свойство делает его полезным в роли уникального идентификатора? | Symbol — примитивный тип, значения которого всегда уникальны. Каждый вызов Symbol() создаёт новый уникальный символ, даже если передать одинаковое описание: Symbol("id") !== Symbol("id"). Это делает Symbol идеальным ключом объекта без риска коллизий с другими свойствами | Symbol — это число с уникальным именем | Два Symbol с одинаковым описанием равны | Symbol используется только в TypeScript

Как узнать длину строки и как обратиться к символу по индексу? | Длину строки возвращает свойство str.length. Обратиться к символу можно через квадратные скобки: str[0] — первый символ (индексация с 0), либо через метод str.charAt(0) | Длину строки возвращает метод str.size() | Для доступа к символу используется str.char(n) | Индексация строк начинается с 1

Что такое NaN и Infinity, для чего Number.isNaN и Number.isFinite, и чем BigInt отличается от number? | NaN (Not a Number) — результат некорректной числовой операции (например, 0/0 или parseInt("abc")). Infinity — результат деления на ноль или переполнения. Number.isNaN(x) проверяет, является ли x именно NaN (без приведения типов). Number.isFinite(x) проверяет конечность числа. BigInt — тип для целых чисел произвольной длины (с суффиксом n: 9007199254740993n), недоступных для number из-за ограничений IEEE 754 | NaN — это ноль; Infinity — строка | isNaN и Number.isNaN работают одинаково | BigInt и number можно смешивать в операциях напрямую

# DOM и события

Что такое DOM и как HTML-страница представлена в виде DOM-дерева? | DOM (Document Object Model) — программный интерфейс, представляющий HTML-документ в виде дерева объектов. Каждый тег — это узел (node), вложенные теги — дочерние узлы, атрибуты и текст тоже являются узлами. Через DOM JavaScript может читать, изменять, добавлять и удалять элементы страницы | DOM — это база данных для хранения HTML | DOM существует только при наличии CSS | DOM — это массив всех тегов страницы

Какими способами можно искать элементы в DOM и чем querySelector отличается от querySelectorAll? | Способы поиска: getElementById, getElementsByClassName, getElementsByTagName, querySelector, querySelectorAll. querySelector(selector) возвращает первый подходящий элемент или null. querySelectorAll(selector) возвращает NodeList всех совпадающих элементов (может быть пустым). Оба принимают CSS-селекторы | querySelector возвращает массив, querySelectorAll — один элемент | querySelectorAll работает только с классами | getElementById быстрее querySelector только для атрибутов

Опишите назначение методов getAttribute, setAttribute, hasAttribute и removeAttribute | getAttribute(name) — читает значение атрибута. setAttribute(name, value) — устанавливает атрибут (или изменяет существующий). hasAttribute(name) — возвращает true/false, есть ли такой атрибут. removeAttribute(name) — удаляет атрибут из элемента | getAttribute удаляет атрибут | setAttribute читает значение атрибута | hasAttribute изменяет значение атрибута

Что такое classList и что делают его методы add, remove, contains и toggle? | classList — объект-коллекция CSS-классов элемента. add(cls) — добавляет класс. remove(cls) — удаляет класс. contains(cls) — возвращает true, если класс есть. toggle(cls) — добавляет класс, если его нет, и удаляет, если есть | classList — это массив атрибутов элемента | toggle добавляет только новые классы, не удаляет | contains возвращает индекс класса в списке

В чём разница между innerHTML и textContent и какой риск возникает при вставке через innerHTML данных от пользователя? | innerHTML читает и устанавливает HTML-разметку внутри элемента (теги интерпретируются). textContent читает и устанавливает только текстовое содержимое (теги экранируются как обычный текст). Риск при innerHTML с пользовательскими данными — XSS-атака (Cross-Site Scripting): злоумышленник может внедрить вредоносный скрипт | innerHTML работает медленнее textContent в 10 раз | textContent может выполнять скрипты | XSS-атаки возможны только при использовании textContent

Какими методами создаются, добавляются и удаляются элементы DOM и для чего нужен элемент template? | Создание: document.createElement(tag). Добавление: parent.append(el), parent.prepend(el), el.before(el2), el.after(el2). Удаление: el.remove(). Элемент <template> хранит HTML-разметку, которая не отображается при загрузке страницы и клонируется по необходимости через template.content.cloneNode(true) | createElement принимает класс CSS | remove() удаляет только текстовые узлы | <template> — это невидимый div

Что такое событие в браузере, каково назначение addEventListener и что содержит объект event? | Событие — сигнал о том, что что-то произошло (клик, нажатие, загрузка). addEventListener(type, handler) подписывает функцию-обработчик на событие. Объект event содержит информацию о событии: type (тип), target (элемент, на котором произошло событие), currentTarget, координаты и др. | addEventListener вешает обработчик только на click | event.target — это весь документ | addEventListener можно вызвать только один раз для одного элемента

Что делает метод preventDefault() и у какого события есть отменяемое поведение по умолчанию? | event.preventDefault() отменяет стандартное поведение браузера для события. Например: у <a> — переход по ссылке, у <form> — отправка формы на сервер, у checkbox — переключение состояния. Вызов preventDefault() позволяет обработать событие кастомным кодом | preventDefault полностью останавливает всплытие события | preventDefault отменяет все обработчики события | preventDefault работает только с событием click

Что такое делегирование событий и какие преимущества оно даёт? | Делегирование событий — техника навешивания одного обработчика на родительский элемент вместо множества обработчиков на каждый дочерний. Работает благодаря всплытию событий. Преимущества: 1) экономия памяти при большом числе элементов; 2) автоматическая обработка динамически добавляемых элементов. Конкретный элемент определяется через event.target | Делегирование отменяет всплытие событий | Делегирование работает только с событием mouseover | Делегирование замедляет обработку событий

Как устроен событийный цикл: стек вызовов, очередь макрозадач и Web APIs? | Стек вызовов (call stack) — выполняет синхронный код по принципу LIFO. Web APIs (setTimeout, fetch и др.) работают вне стека. Когда их колбэк готов, он попадает в очередь макрозадач (task queue). Event loop проверяет: если стек пуст, берёт первую задачу из очереди и помещает в стек | Стек вызовов выполняет асинхронный код первым | Event loop постоянно прерывает синхронный код | Web APIs — это встроенные функции JavaScript, выполняемые в стеке

Как из JavaScript получить доступ к форме, чем отличаются change и input, и чем reset от submit? | Форму можно найти через querySelector('form') или document.forms. Событие input срабатывает при каждом изменении поля (в реальном времени), change — только когда поле теряет фокус или выбор подтверждён. reset() сбрасывает поля формы к начальным значениям, submit() программно отправляет форму | change срабатывает при каждом нажатии клавиши | reset() отправляет форму на сервер | submit() очищает поля формы

В чём разница между localStorage и sessionStorage и какого типа данные в них хранятся? | localStorage хранит данные без ограничения по времени (до явной очистки), доступен во всех вкладках одного домена. sessionStorage существует только в течение сессии (закрытие вкладки удаляет данные) и не передаётся между вкладками. В обоих хранятся только строки — объекты нужно сериализовать через JSON.stringify | localStorage очищается при обновлении страницы | sessionStorage доступен из других доменов | Оба хранилища поддерживают хранение любых типов данных

# Качество кода и инструменты

Что такое ревью кода и почему важны единый стиль и осмысленное именование? | Ревью кода — процесс проверки кода другим разработчиком до его принятия в основную ветку. Единый стиль (форматирование, кавычки, отступы) снижает когнитивную нагрузку: команда читает «одним голосом». Осмысленные имена переменных делают код самодокументированным, уменьшают время на понимание и число ошибок | Ревью кода автоматически исправляет ошибки | Стиль кода важен только для больших проектов | Именование переменных не влияет на читаемость кода

Чем различаются ошибки SyntaxError, ReferenceError и TypeError? | SyntaxError — нарушение синтаксиса языка, обнаруживается при парсинге (например, пропущена скобка). ReferenceError — обращение к необъявленной переменной (например, console.log(x) где x не объявлен). TypeError — операция над значением неподходящего типа (например, вызов null как функции: null()) | ReferenceError возникает при синтаксической ошибке | TypeError — это ошибка компилятора TypeScript | SyntaxError возникает только в асинхронном коде

Чем логические ошибки опаснее синтаксических и как помогает console.log? | Синтаксические ошибки выявляет интерпретатор — код просто не запускается. Логические ошибки не вызывают исключений: программа работает, но выдаёт неверный результат. Их сложнее обнаружить, потому что нет сообщения об ошибке. console.log выводит промежуточные значения переменных, помогая отследить, где логика расходится с ожидаемым | Логические ошибки легче синтаксических, потому что код работает | console.log исправляет логические ошибки автоматически | Синтаксические ошибки опаснее, так как сложнее диагностировать

Что такое инструкция debugger и чем отладка через точки останова удобнее console.log? | Инструкция debugger — при наличии открытых DevTools браузер приостанавливает выполнение в этом месте, открывая отладчик. Точки останова удобнее тем, что можно: видеть все переменные в контексте, пошагово выполнять код (step over/into), изменять значения на лету — без необходимости добавлять и убирать console.log в исходнике | debugger — это метод объекта console | Точки останова работают только в Node.js | console.log всегда информативнее debugger

# Модули и инструменты сборки

Что такое IIFE и какую проблему она помогает решить по сравнению с глобальными переменными? | IIFE (Immediately Invoked Function Expression) — функция, которая объявляется и сразу же вызывается: (function() { ... })(). Переменные внутри IIFE не попадают в глобальную область видимости, что предотвращает загрязнение глобального пространства имён и конфликты имён между скриптами | IIFE — это стрелочная функция без параметров | IIFE используется только для асинхронного кода | IIFE замедляет выполнение кода по сравнению с обычной функцией

Что такое инкапсуляция и как модули в JavaScript помогают её обеспечить? | Инкапсуляция — принцип сокрытия внутренней реализации и предоставления только публичного интерфейса. ES-модули (import/export) обеспечивают инкапсуляцию: всё, что объявлено в модуле, приватно по умолчанию; наружу выходит только то, что явно экспортировано | Инкапсуляция — это запрет на изменение переменных | Модули делают все переменные глобальными | Инкапсуляция относится только к классам

Объясните разницу между именованным экспортом и экспортом по умолчанию и чем отличается синтаксис их импорта | Именованный экспорт: export function foo() {} — можно экспортировать несколько из одного файла; импорт с фигурными скобками: import { foo } from './file'. Экспорт по умолчанию: export default function bar() {} — один на файл; импорт без скобок с любым именем: import myBar from './file' | Экспорт по умолчанию требует фигурных скобок при импорте | Именованный экспорт ограничен одним объектом на файл | При дефолтном экспорте нельзя переименовывать при импорте

Для чего нужны директивы export as и import as и в каких случаях прибегают к переименованию? | export { foo as bar } позволяет экспортировать под другим именем (полезно, если внутреннее имя конфликтует с соглашениями или нежелательно раскрывать). import { foo as myFoo } позволяет избежать конфликта имён при импорте нескольких модулей, у которых есть одинаково названные экспорты | as используется только при переименовании CSS-классов | import as делает переменную константой | export as можно использовать только с дефолтным экспортом

Что такое NPM, за что отвечает package.json и что решают библиотеки date-fns, Axios, Zod и Chart.js? | NPM (Node Package Manager) — менеджер пакетов для JavaScript. package.json — файл-манифест проекта: содержит название, версию, зависимости и скрипты. date-fns — работа с датами. Axios — HTTP-запросы. Zod — валидация и типизация данных (схемы). Chart.js — визуализация данных (графики и диаграммы) | NPM — это редактор кода | package.json хранит исходный код приложения | Axios используется для работы с датами

Что такое сборка кода, чем отличаются Webpack и Vite и что такое tree shaking? | Сборка — преобразование исходных модулей в файлы, готовые к работе в браузере (объединение, минификация, транспиляция). Webpack — зрелый сборщик, конфигурируется через webpack.config.js, медленнее на старте. Vite — современный сборщик, использует нативные ES-модули в разработке (мгновенный старт), оптимизированная сборка через Rollup. Tree shaking — исключение неиспользуемого кода из бандла | Vite — это линтер для TypeScript | Webpack и Vite выполняют одинаковые задачи без отличий | Tree shaking добавляет код из зависимостей автоматически

# Формы и валидация

Какие средства встроенной браузерной валидации форм существуют? Опишите атрибуты required, type, minlength/maxlength и pattern | required — поле обязательно для заполнения. type — определяет тип поля (email, number, url и др.) и добавляет соответствующую валидацию. minlength/maxlength — ограничения на длину текста. pattern — принимает регулярное выражение; значение должно соответствовать шаблону. При нарушении браузер показывает встроенные сообщения об ошибках | required позволяет отправить форму без заполнения | minlength задаёт максимально допустимое значение | pattern проверяет тип данных, а не формат

Что такое объект ValidityState и за что отвечают свойства valueMissing, typeMismatch и patternMismatch? | ValidityState — объект, доступный через input.validity, содержащий булевы свойства состояния валидации. valueMissing — true, если поле required пустое. typeMismatch — true, если значение не соответствует типу (например, в поле email введён не email). patternMismatch — true, если значение не соответствует атрибуту pattern | ValidityState — это объект события формы | valueMissing означает, что тип данных неверный | patternMismatch срабатывает при превышении maxlength

Для чего используется атрибут novalidate и зачем отключать встроенную валидацию при кастомной? | novalidate добавляется к тегу <form> и отключает встроенную браузерную валидацию при отправке. Это нужно, когда разработчик реализует собственную валидацию на JavaScript: встроенные сообщения браузера и кастомные конкурировали бы, возможно, дублируя ошибки или мешая UX | novalidate ускоряет отправку формы | novalidate обязателен при использовании атрибута pattern | novalidate удаляет атрибут required из всех полей

Что такое регулярное выражение и каково назначение якорей ^ и $, наборов [a-z] и квантификаторов *, +, ?? | Регулярное выражение — шаблон для поиска и проверки строк. ^ — начало строки, $ — конец строки. [a-z] — набор символов (любая буква от a до z). Квантификаторы: * — ноль или более повторений, + — одно или более, ? — ноль или одно. Пример: /^[a-z]+$/ проверяет строку из одних строчных латинских букв | ^ означает «не содержит символ» | + означает «ноль или один символ» | $ обозначает начало строки

За что отвечают флаги g, i, m, u в регулярных выражениях, что делают группировка и альтернация? | g (global) — искать все совпадения, не только первое. i (insensitive) — регистронезависимый поиск. m (multiline) — ^ и $ совпадают с началом/концом каждой строки. u (unicode) — корректная работа с Unicode. Группировка () объединяет часть паттерна и позволяет применять к ней квантификаторы. Альтернация | — «или» (cat|dog совпадает с cat или dog). Экранирование \ — трактует спецсимвол как обычный | g означает «только одно совпадение» | u флаг делает поиск регистронезависимым | Группировка () применяется только для именования

# Асинхронность

В чём разница между синхронным и асинхронным кодом и зачем во фронтенде нужен асинхронный код? | Синхронный код выполняется последовательно: каждая строка ждёт завершения предыдущей. Асинхронный код запускает операцию и не блокирует выполнение — продолжает работу дальше. Во фронтенде асинхронность нужна для сетевых запросов, таймеров, работы с файлами — без неё браузер «зависал» бы на каждом запросе к серверу | Асинхронный код выполняется быстрее синхронного | Синхронный код используется только в Node.js | Асинхронный код — это код с ошибками

Что такое колбэк-функция, чем синхронный колбэк отличается от асинхронного? | Колбэк (callback) — функция, передаваемая другой функции в качестве аргумента для последующего вызова. Синхронный колбэк вызывается немедленно внутри вызывающей функции (например, в Array.forEach). Асинхронный колбэк вызывается позже, когда завершится асинхронная операция (например, колбэк в addEventListener или в обработчике загрузки изображения: img.onload = callback) | Колбэк — это функция без параметров | Асинхронный колбэк выполняется до синхронного | Колбэк нельзя передавать в стрелочные функции

Сравните setTimeout и setInterval: что они возвращают и как остановить выполнение? | setTimeout(fn, ms) — выполняет fn один раз через ms миллисекунд, возвращает числовой идентификатор таймера. Остановить: clearTimeout(id). setInterval(fn, ms) — выполняет fn повторно каждые ms миллисекунд, возвращает id. Остановить: clearInterval(id) | setTimeout выполняется бесконечно | setInterval выполняется ровно один раз | Оба функции останавливаются автоматически после первого вызова

Как Event Loop выполняет асинхронный код и почему setTimeout с задержкой 0 сработает не мгновенно? | Когда синхронный код помещает setTimeout в Web API, по истечении задержки колбэк помещается в очередь макрозадач. Event loop переносит задачу в стек только когда стек полностью пуст. Поэтому setTimeout(fn, 0) выполнится только после завершения всего текущего синхронного кода — задержка 0 значит «как можно скорее, но не сейчас» | setTimeout(fn, 0) выполняется до любого синхронного кода | Event loop прерывает синхронный код при появлении таймера | Очередь макрозадач обрабатывается параллельно со стеком

Что такое Promise и каковы его три состояния? | Promise — объект, представляющий результат асинхронной операции, которая может завершиться в будущем. Три состояния: 1) pending — ожидание (начальное состояние); 2) fulfilled — успешно выполнен (вызван resolve); 3) rejected — завершён с ошибкой (вызван reject). Переход из pending необратим | Promise может находиться в нескольких состояниях одновременно | Четвёртое состояние Promise — cancelled | fulfilled и resolved — разные состояния промиса

Как строится цепочка .then()/.catch() и для чего нужен Promise.all? | .then(fn) вызывается при успешном выполнении и возвращает новый промис, что позволяет строить цепочку. .catch(fn) обрабатывает ошибку. Promise.all([p1, p2]) ждёт выполнения всех промисов и возвращает массив результатов. Если хотя бы один промис отклоняется — Promise.all немедленно переходит в rejected и вызывается .catch | .then() не возвращает новый промис | .catch() перехватывает только ошибки синтаксиса | Promise.all ждёт только первого выполнившегося промиса

# HTTP и fetch

Что такое протокол HTTP и из каких частей состоят HTTP-запрос и HTTP-ответ? | HTTP — протокол передачи данных между клиентом и сервером. Запрос: стартовая строка (метод + URL + версия), заголовки (headers), тело (body, опционально). Ответ: статусная строка (версия + код + текст), заголовки (Content-Type и др.), тело (HTML, JSON и т.д.) | HTTP — протокол только для передачи изображений | HTTP-запрос не содержит заголовков | HTTP-ответ состоит только из статус-кода

В чём разница между GET и POST, что такое MIME Type и что означают статус-коды 2xx, 4xx, 5xx? | GET — запрашивает данные, параметры в URL, без тела. POST — отправляет данные на сервер, данные в теле запроса. MIME Type — идентификатор типа данных (text/html, application/json, image/png). Статус-коды: 2xx — успех (200 OK, 201 Created), 4xx — ошибка клиента (404 Not Found, 400 Bad Request), 5xx — ошибка сервера (500 Internal Server Error) | GET передаёт данные в теле запроса | 4xx означает ошибку на сервере | 2xx означает ошибку клиента

Что такое метод fetch и формат JSON, и каково назначение JSON.stringify, JSON.parse и response.json()? | fetch(url) — встроенный метод браузера для HTTP-запросов, возвращает Promise. JSON (JavaScript Object Notation) — текстовый формат обмена данными. JSON.stringify(obj) — преобразует объект в JSON-строку. JSON.parse(str) — парсит JSON-строку в объект JavaScript. response.json() — читает тело ответа и парсит его как JSON, возвращает Promise | fetch — синхронная функция | JSON.parse преобразует объект в строку | response.json() возвращает строку, а не объект

Что такое токены и эмбеддинги при работе языковой модели и за счёт чего модель формирует ответ? | Токены — единицы текста, на которые разбивается входной текст (слова, подслова, символы). Эмбеддинги — числовые векторы, представляющие токены в многомерном пространстве, где близкие по смыслу слова находятся рядом. Модель формирует ответ, предсказывая следующий наиболее вероятный токен на основе весов, полученных при обучении | Токены — это отдельные символы Unicode | Эмбеддинги — это сжатые изображения | Модель выбирает слова из заранее составленного списка ответов

# TypeScript

Что такое динамическая типизация в JavaScript и какие проблемы порождает слабая типизация? | Динамическая типизация — тип переменной определяется в процессе выполнения, не фиксируется при объявлении. Слабая типизация — JavaScript автоматически приводит типы в операциях. Проблемы: неочевидные баги (например, "5" - 2 === 3, но "5" + 2 === "52"), трудно обнаруживаемые ошибки, хаос в больших кодовых базах | Динамическая типизация — это типизация только во время компиляции | Слабая типизация предотвращает все ошибки | В JavaScript тип переменной нельзя изменить после объявления

Что такое JSDoc и как он помогает сделать код самодокументированным? | JSDoc — стандарт документирующих комментариев в формате /** ... */. Позволяет описывать типы параметров (@param {number} x), возвращаемого значения (@returns {string}), назначение функции. Редакторы (VS Code) используют JSDoc для автодополнения и подсказок типов без перехода на TypeScript | JSDoc — это фреймворк для тестирования | JSDoc компилирует JavaScript в TypeScript | JSDoc используется только в Node.js проектах

Какие средства проверки типов во время исполнения есть в JavaScript? | typeof — возвращает строку с типом примитива ("number", "string", "boolean", "undefined", "object", "function"). Array.isArray(x) — проверяет, является ли x массивом (typeof [] возвращает "object", поэтому нужна отдельная функция). instanceof — проверяет принадлежность объекта к цепочке прототипов конструктора (x instanceof Date) | typeof возвращает числовой код типа | Array.isArray() проверяет тип всех коллекций | instanceof работает только со встроенными типами

Что нужно для добавления TypeScript в проект, какова роль tsconfig.json и что такое компиляция TS в JS? | Установка: npm install typescript --save-dev. tsconfig.json — файл конфигурации TypeScript компилятора: задаёт версию целевого JS (target), папки с исходниками (include), строгость проверки (strict) и др. Компиляция (tsc) — преобразование .ts-файлов в .js: TypeScript проверяет типы и удаляет аннотации, оставляя чистый JavaScript для браузера | TypeScript работает в браузере без компиляции | tsconfig.json хранит зависимости проекта | Компиляция TypeScript добавляет проверку типов в рантайм

Что такое строгий режим (strict) в TypeScript и что задают опции include и exclude? | strict: true в tsconfig.json включает набор строгих проверок: noImplicitAny (нельзя оставлять неявный any), strictNullChecks (null/undefined не присваиваются другим типам без явной проверки) и др. include — массив паттернов путей к файлам, которые TypeScript должен компилировать. exclude — паттерны файлов/папок, которые нужно исключить (например, "node_modules") | strict отключает проверку типов | include указывает путь к tsconfig.json | exclude блокирует установку пакетов

Что такое приведение типов через as, чем оно отличается от настоящей проверки типа и какие риски несёт? | as — утверждение типа: сообщает TypeScript «доверься мне, это значение такого типа» (const x = value as string). Это только указание компилятору, реальная проверка в рантайме не выполняется. Риск: если тип неверный, в рантайме возникнут ошибки, которые TypeScript не предупредил | as выполняет реальное приведение типа в рантайме | as — синоним оператора typeof | as можно использовать только с примитивами

Что такое защитники типов (type guards) и чем кастомный защитник отличается от typeof/instanceof? | Type guard — выражение, которое сужает тип переменной в определённой ветке кода. Встроенные: typeof x === "string", x instanceof Date — работают для примитивов и классов. Кастомный type guard — функция с предикатом типа в возвращаемом значении: function isUser(x: any): x is User { return ... }. Позволяет проверять произвольные структуры данных | Type guards — это декораторы TypeScript | Кастомный type guard автоматически генерируется компилятором | typeof проверяет принадлежность к пользовательским интерфейсам

Что такое дженерики (обобщённые типы), зачем они нужны? | Дженерики позволяют писать функции и структуры данных, работающие с разными типами без потери информации о типе. Вместо any (который «забывает» тип) используется параметр типа <T>: function identity<T>(x: T): T { return x }. Позволяет использовать одну функцию для разных типов при сохранении типовой безопасности | Дженерики — это обобщённые переменные | Дженерики заменяют интерфейсы в TypeScript | Дженерики работают только с массивами

Для чего используется extends в дженериках и как задать значение типа по умолчанию? | extends в дженериках ограничивает допустимые типы: function foo<T extends object>(x: T) — T может быть только объектом. JSX не является HTML — он транспилируется Babel в вызовы React.createElement(). При изменении состояния React строит новое VDOM, сравнивает с предыдущим (reconciliation/diffing), находит минимальный набор изменений и обновляет только изменившиеся части реального DOM. В функциональных компонентах state управляется через хук useState | props можно изменять напрямую внутри компонента | state передаётся от родителя к дочернему компоненту | Изменение props вызывает перерисовку компонента

Создаётся через React.createContext(), данные предоставляются через <Context.Provider value={...}>, потребляются через useContext(Context) в любом дочернем компоненте | Context API — это роутер React | Context API заменяет state в компонентах | Context API доступен только в классовых компонентах
